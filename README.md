## О проекте

Это **Telegram‑бот** вокруг идеи «одно окно — много нейросетей»: пользователь не прыгает между сайтами провайдеров, а живёт в привычном чате. Внутри — **баланс токенов** (или эквивалент учёта расхода), сценарии по **кнопкам и вложениям**, отдельные ветки для **чата, картинок, видео и аудио**, плюс **профиль, подписка и вспомогательные разделы** (платежи, рефералы, помощь — по задумке продукта).

Интерфейс сознательно **телеграм‑нативный**: промпты текстом, референсы картинками, шаги уточнения параметров (модель, длительность, формат, версия API и т.д.) без отдельного веб‑кабинета как обязательного входа.

---

## Задача с точки зрения продукта

Свести к одному боту:

- **разные классы задач** (диалог, визуал, видео, звук);
- **разных поставщиков моделей** с разной ценой и качеством;
- **понятный учёт расхода** для пользователя («сколько ещё запросов примерно осталось»).

Технически за этим стоят отдельный **HTTP API**, **БД**, **кэш/очередь** и асинхронные сценарии ожидания результата от внешних API — без раскрытия деталей интеграций в этом репозитории.

## Функциональность (по модулям)

### Старт и навигация

Единая точка входа: **стартовое меню** с приветствием, отображением баланса и **сеткой разделов** (чат, фото, видео, аудио, профиль, подписка и связанные сценарии). Пользователь не теряется: каждый раздел ведёт по цепочке кнопок к конкретному действию.

### Чат

Подключены и согласованы в одном UX несколько **чат‑моделей** (в т.ч. **ChatGPT, Grok, Gemini, Deepseek**): выбор модели и параметров — внутри бота, ответы приходят в тот же диалог.

### Фото

Несколько **движков генерации и обработки** (в т.ч. **Seedream, Nano Banana, GPT Image, Midjourney, замена лиц, Grok Image** и др. по конфигурации заказчика). Пользователь выбирает модель, при необходимости прикладывает изображение, задаёт промпт — результат возвращается в Telegram.

### Видео

Линейка **видеогенераторов** (в т.ч. **Kling, Hailuo, Sora 2** и другие подключаемые провайдеры). Для разных движков — **свои параметры**: длительность, соотношение сторон, версия модели, опционально **автоперевод промпта**, загрузка референс‑кадра, ограничения по контенту (например, лица на референсе) — в зависимости от правил API.

### Аудио

Сценарии вроде **создания треков**, **расшифровки голосовых**, **озвучки текста** — выбор режима из меню, дальше бот ведёт по шагам.

### Профиль и монетизация

**Профиль** показывает баланс токенов и **оценку «на сколько запросов хватит»** по разным сервисам (чтобы пользователю было интуитивно, а не только сухие цифры). Отдельно — сценарии **оплаты, рефералов, помощи** и админ‑логика (без деталей в публичном описании).

### Backend (обобщённо)

За ботом стоят **REST API**, **PostgreSQL** для состояния пользователей и биллинга, **Redis** для кэша и фоновых задач, **миграции схемы** и интеграции с **внешними AI‑ и платёжными API**. Конкретные эндпоинты, схемы таблиц и ключи в этот репозиторий не входят.

## Технологии

| Категория | Стек |
|-----------|------|
| Язык | Python 3.11+ |
| Telegram | Aiogram 3 |
| API | FastAPI |
| Данные | PostgreSQL, SQLAlchemy, Alembic |
| Очереди / кэш | Redis |
| Запуск (у заказчика) | Docker Compose (бот + API + БД + Redis) |

Стек отражает типичный **production‑подобный** набор: явное разделение процесса бота и API, версионируемая схема БД, кэш для нагрузки и повторных запросов.

> **Про блок «Languages» на GitHub:** график языков может учитывать **историю коммитов** в репозитории. В **текущем дереве** `main` после витринной очистки остаются в основном Markdown и изображения; визуальная статистика со временем подтянется к актуальному состоянию.
